## What Makes Sticky Layouts Special?

Unlike traditional fixed positioning, sticky layouts create a natural reading flow where important content stays visible while supplementary information scrolls past. This pattern is perfect for:

- **Documentation sites** where code examples stay visible alongside explanations
- **Product showcases** where images remain in focus during feature descriptions
- **Tutorial content** where visual aids complement step-by-step instructions
- **Dashboard layouts** where key metrics stay accessible during data exploration

The magic happens when sticky positioning feels natural and enhances rather than disrupts the user experience.

## The Problem with Traditional Approaches

Before `StickyWrapper`, creating sticky layouts required:

1. **Manual CSS positioning** with complex media queries
2. **Hardcoded HTML structures** that couldn't adapt to different content
3. **JavaScript scroll listeners** that impacted performance
4. **Separate components** for each sticky scenario

This led to maintenance nightmares and limited flexibility. What if you wanted to switch which side content sticks to? What if you had mixed content types? The traditional approach fell apart quickly.

```tsx
interface StickyWrapperProps {
  children: ReactNode;
  side?: "left" | "right";
  className?: string;
  stickyClassName?: string;
  contentClassName?: string;
  stickyOffset?: string;
}

export function StickyWrapper({
  children,
  side = "left",
  stickyOffset = "top-8",
}: StickyWrapperProps) {
  const childrenArray = Array.isArray(children)
    ? children : [children];

  // Find sticky content marked with StickyContent component
  let stickyContent: ReactNode = null;
  let scrollingContent: ReactNode[] = [];

  childrenArray.forEach((child) => {
    if (
      child &&
      typeof child === "object" &&
      "type" in child &&
      child.type === StickyContent
    ) {
      stickyContent = child;
    } else {
      scrollingContent.push(child);
    }
  });

  // Auto-detect: first child as sticky if multiple
  if (!stickyContent && childrenArray.length > 1) {
    stickyContent = childrenArray[0];
    scrollingContent = childrenArray.slice(1);
  }

  return (
    <div className="flex flex-col lg:flex-row gap-8">
      {side === "left" ? (
        <>
          <div className="flex-1">
            <div className={`lg:sticky lg:${stickyOffset}`}>
              {stickyContent}
            </div>
          </div>
          <div className="flex-1">
            {scrollingContent}
          </div>
        </>
      ) : (
        <>
          <div className="flex-1">
            {scrollingContent}
          </div>
          <div className="flex-1">
            <div className={`lg:sticky lg:${stickyOffset}`}>
              {stickyContent}
            </div>
          </div>
        </>
      )}
    </div>
  );
}
```

## The StickyWrapper Solution

The `StickyWrapper` component solves these problems with an elegant, flexible API. Here's what makes it special:

### Automatic Content Detection

The component automatically identifies what should stick by looking for the `StickyContent` component:

```jsx
<StickyWrapper>
  <StickyContent>This content will stick</StickyContent>
  <ScrollContent>
    <div>This content will scroll</div>
    <div>More scrolling content</div>
  </ScrollContent>
</StickyWrapper>
```

### Flexible Side Control

Switch between left and right positioning with a simple prop:

```jsx
// Sticky content on the left
<StickyWrapper side="left">
  <StickyContent>Stays left</StickyContent>
  <ScrollContent>Scrolls right</ScrollContent>
</StickyWrapper>

// Sticky content on the right
<StickyWrapper side="right">
  <StickyContent>Stays right</StickyContent>
  <ScrollContent>Scrolls left</ScrollContent>
</StickyWrapper>
```

### Smart Fallbacks

If no `StickyContent` component is found, the component intelligently uses the first child as sticky content when multiple children are present.

### Responsive Design

The sticky behavior automatically adapts to screen size - sticky on large screens, stacked on mobile devices.

## Advanced Styling and Customization

The `StickyWrapper` provides multiple styling hooks for complete customization:

### CSS Classes

- `className`: Controls the main wrapper styling
- `stickyClassName`: Targets the sticky content container
- `contentClassName`: Styles the scrolling content area

### Sticky Positioning

The `stickyOffset` prop controls how far from the top the sticky content sits:

```jsx
<StickyWrapper stickyOffset="top-16">
  {/* Content sticks 4rem from top */}
</StickyWrapper>

<StickyWrapper stickyOffset="top-32">
  {/* Content sticks 8rem from top */}
</StickyWrapper>
```

### Full-Width Layouts

The component automatically handles full-width layouts that break out of content containers, perfect for immersive experiences.

### Mobile Considerations

On mobile devices, the sticky behavior is disabled and content stacks vertically for optimal reading experience.

## Best Practices

### 1. Content Balance

Ensure sticky and scrolling content have appropriate proportions:

- **50/50 split** works well for most layouts
- **33/67 split** when sticky content is supplementary
- **67/33 split** when sticky content is primary focus

### 2. Mobile Experience

Always test on mobile devices where sticky behavior is disabled:

- Ensure content order makes sense when stacked
- Verify images and code blocks remain readable
- Check that calls-to-action are appropriately positioned

### 3. Performance Considerations

- Use appropriately sized images in sticky content
- Avoid complex animations in sticky sections
- Test scroll performance on lower-end devices
- Monitor layout shift during initial render

### 4. Accessibility Guidelines

- Maintain logical heading hierarchy
- Ensure sufficient color contrast in all sections
- Provide alternative text for sticky images
- Test keyboard navigation thoroughly
