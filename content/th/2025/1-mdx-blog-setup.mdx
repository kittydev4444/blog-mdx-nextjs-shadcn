MDX รวมพลังของ Markdown เข้ากับความยืดหยุ่นของ React components ทำให้เหมาะสำหรับบล็อกทางเทคนิคและเว็บไซต์เอกสาร

## โครงสร้างโปรเจกต์

บล็อกของเราเป็นไปตามโครงสร้างที่สะอาดและเป็นระเบียบ:

```tree
content/
├── en/
|   ├── 2024/
|   |   ├── 1-getting-started-with-nextjs.mdx
|   |   └── 2-building-with-shadcn-ui.mdx
|   └── 2025/
|       └── 1-mdx-blog-setup.mdx
└── th/
    ├── 2024/
    |   ├── 1-getting-started-with-nextjs.mdx
    |   └── 2-building-with-shadcn-ui.mdx
    └── 2025/
        └── 1-mdx-blog-setup.mdx
```

## คุณสมบัติ

### การค้นหาเนื้อหาอัตโนมัติ

ระบบจะค้นหาไฟล์ MDX ทั้งหมดในไดเรกทอรีเนื้อหาโดยอัตโนมัติและจัดระเบียบตามปีและภาษา

### Dynamic Routing

แต่ละโพสต์บล็อกจะได้ URL เป็นของตัวเองตามชื่อไฟล์ (ลบส่วนนำหน้าลำดับออก):

- `1-getting-started-with-nextjs.mdx` → `/blog/getting-started-with-nextjs`

### การรองรับ Frontmatter

แต่ละไฟล์ MDX รวมถึงข้อมูลเมตาใน frontmatter:

```js
---
title: "ชื่อโพสต์ของคุณ"
date: "2025-01-15"
excerpt: "คำอธิบายสั้นๆ ของโพสต์ของคุณ"
---
```

## ประโยชน์ของ MDX

### 1. ความเรียบง่ายของ Markdown

เขียนเนื้อหาด้วยไวยากรณ์ Markdown ที่คุ้นเคย

### 2. การรวม React Component

ฝัง React components แบบโต้ตอบได้โดยตรงในเนื้อหาของคุณ:

```jsx
<Button onClick={() => alert("สวัสดี!")}>คลิกฉัน!</Button>
```

### 3. Syntax Highlighting

โค้ดบล็อกจะได้รับ syntax highlighting อัตโนมัติเพื่อการอ่านที่ดีขึ้น

## การเริ่มต้น

1. **สร้างเนื้อหาของคุณ**: เพิ่มไฟล์ MDX ไปยังโฟลเดอร์ปีและภาษาที่เหมาะสม
2. **เพิ่ม frontmatter**: รวมชื่อเรื่อง วันที่ และคำอธิบาย
3. **เริ่มเขียน**: ใช้ไวยากรณ์ Markdown กับ React components ตามต้องการ

## สรุป

ระบบบล็อกที่ขับเคลื่อนด้วย MDX นี้ให้ความสมดุลที่สมบูรณ์แบบระหว่างความเรียบง่ายและพลัง ช่วยให้คุณมุ่งเน้นไปที่เนื้อหาในขณะที่ยังคงควบคุมการนำเสนออย่างเต็มที่
